Asukaflight is a highly customizable program and accompanying firmware patch
which allows mouse or keyboard input to drive the control sticks on a
Radiomaster GX12 radio. It can be used for FPV drones, other remotely
controlled vehicles, and simulators, and it can use radio gimbal input on one
stick with mouse input on the other.

The trainer port is not used. All I/O runs over the USB-C port, which avoids
the low polling rate of the physical trainer port.

Asukaflight also supports USB joystick output so the same setup can be tested in
simulators such as VelociDrone.

Support is eventually planned for USB joysticks and other radios, especially
the TX16S family. Current packages may include experimental TX16S firmware
candidates, but they are untested and separate from the
validated GX12 firmware. Flash at your own peril, although they shouldn't
cause any issues beyond potentially not working with Asukaflight,
and from what I've read, DFU radios can always be recovered if something goes wrong.
To discuss hardware support and/or send a radio to the
developer's PO box, contact `arconautics@gmail.com`.

Asukaflight supports mice and other USB devices with polling rates up to
8000 Hz. The firmware patch raises the EdgeTX mixer path to 1000 Hz and doubles
input precision during mouse mode.

Asukaflight includes input shaping algorithms for customizing control feel. The
default profile uses elastic return shaping for a familiar mouse-control feel,
and advanced profiles can use expo controls and graph-node shaping.

Additional features:

- Wooting analog API support for Wooting keyboards.
- Dual-mouse input for driving the left and right sticks with separate mice.

Planned features:

- Expanded support for more radios and USB HID devices.
- Recording and playback of inputs.
- (maybe) Optional remote input over the Internet.

The current tested path is:

```text
GameInput mouse capture -> gx12mouse.exe -> composite USB CDC SBUS
  -> GX12 EdgeTX trainer/mixer -> GX12 HID joystick or ExpressLRS RF
```

Release users should use the firmware-included zip generated by
`tools\publish-gx12-distribution.ps1` and run `Asukaflight.exe` from the
extracted folder. Keep the folder together - the launcher expects `profiles\`,
`runtime\gx12mouse.exe`, and `firmware\` beside the app in release
packages.

## User Directions

To set up a Radiomaster GX12 model for Asukaflight, first download the latest .zip from the Releases section and extract. Then:

1. Power on the radio in bootloader mode by holding the horizontal trim buttons
   (T1/T4) together.
2. Plug the radio into the computer and copy `firmware\R2X-871.BIN` from the
   extracted package to the radio's `FIRMWARE` folder.
3. Back up models and current firmware before flashing if you wish.
4. Unplug USB, choose `Write Firmware`, and select `R2X-871.BIN`.
5. After flashing and rebooting, plug in USB and select USB Joystick mode.
6. Under `MDL > 2/12 SETUP`, set `Trainer - Mode` to `Master/Serial`.
7. Under `MDL > 10/12 SPECIAL FUNCTIONS`, create an entry for an unused switch,
   for example `SW2 DOWN`.
8. On that entry, set `Trainer - Axis` and enable the checkmark.
9. Under `SYS > 5/7 TRAINER`, set the gimbal axes intended for Asukaflight to
   `+=` or `:=` mode, for example:

```text
Ail - += - 100 CH1
Ele - += - 100 CH2
Thr - += - 100 CH3
Rud - += - 100 CH4
```

10. Start `Asukaflight.exe` and, with the GX12 still in USB Joystick mode,
    select `Start Composite Trainer`.

When the mouse moves, the GX12 right gimbal monitor should show movement on the
radio home-screen monitor page. The default profile uses the GX12 left gimbal
for throttle/yaw and the mouse for roll/pitch. `F3` freezes mouse movement, and
`F1` stops the trainer runtime.

Test first in a simulator or with a harmless vehicle eg. a tinywhoop. Do not use
this program where loss of control could harm people or property.

If flashing fails, restore stock firmware with EdgeTX Buddy or the relevant
EdgeTX recovery tools. Keep the radio powered during firmware updates.

## Repository Layout

- `src\main.cpp` - native trainer/runtime executable.
- `apps\Gx12.Launcher.Wpf` - primary WPF launcher, published as
  `Asukaflight.exe`.
- `firmware\R2X-871.BIN` - validated GX12 firmware image.
- `firmware\edgetx-gx12-2.11.0` - EdgeTX-derived GX12 firmware source.
- `firmware\experimental-tx16s` - untested TX16S/TX16S MK3 tester firmware.
- `profiles\whoop-linear.toml` - current default profile.
- `tools\publish-gx12-distribution.ps1` - firmware-included public package
  builder.
- `tools\publish-gx12-release.ps1` - internal portable app publisher used by
  the distribution builder.
- `tests\Gx12.Launcher.Tests` - launcher and packaging regression tests.
- `docs\release-packages.md` - portable package and distribution build notes.
- `docs\firmware-source.md` - firmware rebuild recipe and validated hash.
- `docs\licensing.md` - GPLv2 and third-party licensing notes.

## Development Checks

From the repo root:

```powershell
dotnet build apps\Gx12.Launcher.Wpf\Gx12.Launcher.Wpf.csproj
dotnet run --project tests\Gx12.Launcher.Tests\Gx12.Launcher.Tests.csproj
dotnet apps\Gx12.Launcher.Wpf\bin\Debug\net7.0-windows\Asukaflight.dll --self-test
```

The native runtime build uses CMake plus the pinned FetchContent dependencies in
`CMakeLists.txt`. GameInput support is enabled when the Microsoft GameInput
NuGet payload is restored under `tool-cache\nuget\Microsoft.GameInput`.

## Release Package

Build the flash-and-run Windows package from the repo root:

```powershell
pwsh -NoProfile -ExecutionPolicy Bypass -File tools\publish-gx12-distribution.ps1
```

The script expects a current native runtime at
`runtime\gx12mouse.exe`, publishes the WPF app, copies profiles and
fixed UI sprites, strips `.pdb` files, runs the packaged self-test, adds the
validated GX12 firmware as `firmware\R2X-871.BIN`, flash/run instructions, and
a SHA256 manifest, then creates a flattened zip such as
`dist\Asukaflight-0.9.0-p2-win-x64.zip`.
Experimental TX16S/TX16S MK3 firmware candidates, when present, are packaged
under `firmware\TX16S-EXPERIMENTAL\` with separate warnings.

## Safety

Before any real RF use after a package or tune change, recheck physical throttle
cut, GX12 trainer routing, stale-trainer/disconnect behavior, stop/freeze keys,
and props-off behavior.

Test first in a simulator or with a harmless vehicle eg. a tinywhoop. Do not use
this program where loss of control could harm people or property.
